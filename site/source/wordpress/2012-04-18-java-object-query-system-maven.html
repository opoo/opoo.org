上一篇介绍很久前的开源项目JOQS，最近又将其源码结构进行了整理，新目录结构采用maven2默认的工程目录结构，整理后的源码发布在本站的SourceForge SVN库中。

整理后的项目不再推荐在SourceForge.net或者Google Code上下载，取而代之的是配置maven 的repository为本站的库，要使用JOQS的包，只需要在您项目的pom.xml中增加以下配置即可。 

     ...
     <dependencies>
        ...
        <dependency>
            <groupId>org.opoo</groupId>
            <artifactId>joqs</artifactId>
            <version>1.0-GA</version>
        </dependency>
    </dependencies>
    ...
    <repositories>
        <repository>
            <id>opoo.org</id>
            <name>opoo.org repository</name>
            <url>http://opoo.org/maven/releases</url>
        </repository>
    </repositories>
    ...

JOQS示例工程

JOQS示例工程具体演示了JOQS的配置和使用方法，可通过以下指令从SVN库中检出示例工程的源代码。 
${'$'} svn checkout https://joqs.svn.sourceforge.net/svnroot/joqs/samples/ joqs-samples
运行如下指令创建Eclipse工程，并将工程导入在Eclipse中。 
${'$'} mvn eclipse:eclipse -DdownloadSources=true -DdownloadJavadocs=true
运行showceses/db/hsqldb/目录中的server.sh(Linux)或者server.bat(Windows)启动数据库，之后在Eclipse中打开Main.java运行即可。
